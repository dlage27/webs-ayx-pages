# webs-ayx-pages

本项目用于归档和发布多个独立的 HTML 页面，不针对任何特定域名或网站。仓库内的页面可作为静态资源直接访问，适合个人或小型项目进行页面存档、分享或快速预览。

## 项目简介

这是一个轻量级的页面归档仓库，主要用于存放独立的 HTML 文件及其相关资源。每个页面均可独立运行，不依赖后端服务。仓库结构简单，便于管理多个不相关的页面版本。

## 目录结构

```
webs-ayx-pages/
├── index.html          # 仓库首页或导航页
├── pages/              # 存放各个独立页面的子目录
│   ├── example1/
│   │   ├── index.html
│   │   └── assets/
│   ├── example2/
│   │   ├── index.html
│   │   └── assets/
│   └── ...
└── README.md           # 本文件
```

- `index.html`：仓库入口页面，可列出所有归档页面的链接或简介。
- `pages/`：每个子目录代表一个独立页面，包含其 HTML、CSS、JavaScript 及静态资源。
- 各页面资源尽量保持自包含，避免跨目录引用。

## 页面归档说明

- 每个归档页面均为独立 HTML 文件，可直接在浏览器中打开运行。
- 页面可能包含图片、样式表、脚本等资源，均存放于对应子目录的 `assets/` 文件夹内。
- 不保证所有页面兼容所有浏览器，建议使用现代浏览器查看。
- 部分页面可能为早期版本或实验性内容，可能存在未优化的代码。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来改进页面内容或修复问题。
- 新增页面请遵循现有目录结构，保持资源自包含。
- 如需删除或更新页面，请确保不影响其他归档页面的引用。
- 本仓库不涉及商业用途，仅用于技术展示与归档。
